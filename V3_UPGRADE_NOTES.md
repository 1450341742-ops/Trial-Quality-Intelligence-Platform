# V3 Upgrade Notes

本次升级新增 `app_v3.py`，继续保留 `app.py` 和 `app_v2.py`，不破坏原有稳定版本。

## V3核心升级目标

V3不只是功能增加，而是将产品从“可演示MVP”升级为更接近申办方高端产品形态的版本：

- 更高级的页面视觉
- 管理层驾驶舱
- 执行驾驶舱
- 中心画像
- 风险热力图
- 证据矩阵
- 核查问答
- 任务中心
- 报告中心
- CAPA中心

## 运行方式

```bash
pip install -r requirements.txt
streamlit run app_v3.py
```

默认账号：

- 系统管理员：admin / admin123
- 申办方QA负责人：qa / qa123
- 项目经理PM：pm / pm123

## V3新增页面

### 1. 管理层驾驶舱

面向老板、QA负责人、注册负责人和项目高层汇报。

展示：

- 项目总数
- 核查准备评分
- 高风险问题数量
- CAPA待关闭数量
- 待办任务数量
- 管理层结论
- TOP风险中心
- 问题分类占比

### 2. 执行驾驶舱

面向PM、QA、CRA、项目执行团队。

展示：

- 项目核查准备评分
- 风险补救建议
- 中心风险排名
- 问题新增趋势
- 关键领域风险分布

### 3. 风险分析

新增四个子视图：

- 中心画像
- 风险热力图
- 高风险问题
- 证据缺口

其中“风险热力图”按“中心 × 问题分类”展示风险分，适合快速定位系统性问题。

### 4. 证据矩阵

根据问题分类、风险等级和证据缺口，自动形成核查前证据准备矩阵。

适合用于：

- CFDI/FDA核查准备
- 注册申报前质量体检
- 中心文件夹补救
- 项目质量专项复核

### 5. CAPA中心

增强CAPA审核展示，突出：

- CAPA质量评分
- 审核结论
- 根因缺口
- 证据缺口
- 预防措施缺口
- 有效性验证缺口

### 6. 报告中心

生成Word综合报告，包含：

- 管理层结论
- 核查准备评分
- 方案关键风险
- 高风险问题清单
- 中心风险排名
- 核查访谈问答
- 核查前行动建议

## V3增强的数据分析能力

### 中心画像

按中心统计：

- 风险分
- 问题数量
- 高风险问题数量
- 问题分类结构

### 风险热力图

按中心和问题分类交叉展示风险分，快速识别高风险中心和高频问题领域。

### 证据缺口识别

系统根据问题分类自动提示应准备的证据，例如：

- 知情同意：伦理批件、ICF版本清单、签署页、筛选检查时间线
- AE/SAE：原始病历、AE/SAE表、上报邮件/系统截图、医学判断记录
- 数据完整性/主要终点：原始记录、EDC截图、Query记录、终点评价表
- 试验用药品：药品台账、温度记录、发放回收记录、偏差评估
- 入排标准：筛选检查报告、入排判断记录、研究者确认、随机化记录

## 推荐演示路径

1. 运行 `streamlit run app_v3.py`
2. 使用 admin/admin123 登录
3. 进入“项目管理”创建项目
4. 进入“文件解析”上传 `sample_data/sample_protocol.txt`
5. 进入“问题清单”上传 `sample_data/sample_findings.csv`
6. 进入“管理层驾驶舱”查看全局管理结论
7. 进入“执行驾驶舱”查看项目执行风险
8. 进入“风险分析”查看中心画像和风险热力图
9. 进入“证据矩阵”查看核查前证据缺口
10. 进入“CAPA中心”审核CAPA质量
11. 进入“核查问答”生成访谈问答
12. 进入“任务中心”生成核查前行动项
13. 进入“报告中心”导出Word综合报告
