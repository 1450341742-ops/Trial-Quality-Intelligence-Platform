# V2 Upgrade Notes

本次升级新增 `app_v2.py`，不覆盖原有 `app.py`，便于保留稳定演示版。

## V2新增能力

1. 登录与角色权限

默认账号：

- 系统管理员：admin / admin123
- 申办方QA负责人：qa / qa123
- 项目经理PM：pm / pm123

角色权限已按系统管理员、QA负责人、PM、注册负责人、只读用户进行预设。

2. PDF解析

新增 PyMuPDF 依赖，支持可复制文本型PDF的正文提取。扫描PDF暂未接入OCR。

3. 文件解析增强

支持 docx、txt、md、pdf 文本提取，并保存提取文本片段到本地SQLite数据库。

4. 风险驾驶舱增强

新增：

- 核查准备风险雷达图
- 严重程度分布
- 中心风险排名
- 问题分类分布
- 高风险问题统计

5. CAPA审核增强

在原有“加强培训、加强管理、已整改”等低质量表述识别基础上，新增：

- 根因分析检查
- 完成证据检查
- 预防措施检查
- 有效性验证检查

6. 核查问答生成

基于项目风险和高风险问题自动生成：

- 申办方QA问答
- PM问答
- 研究者/中心人员问答
- 需准备证据清单

7. 核查前任务清单

可基于当前风险自动生成任务：

- 高风险问题复核任务
- CAPA关闭跟进任务
- 核查准备缺口补救任务

支持导出CSV。

8. 操作日志

记录登录、文件上传、问题导入、任务生成、报告导出、AI配置保存等关键操作。

9. AI接口预留

系统设置中增加AI模型配置入口，可预留 OpenAI、Azure OpenAI、DeepSeek、通义千问、私有化模型等配置。当前默认仍使用本地规则引擎，避免演示依赖外部API。

## V2运行方式

```bash
pip install -r requirements.txt
streamlit run app_v2.py
```

## 推荐演示流程

1. 使用 admin / admin123 登录。
2. 进入“项目管理”，创建项目。
3. 进入“文件解析”，上传 `sample_data/sample_protocol.txt`。
4. 进入“问题清单”，上传 `sample_data/sample_findings.csv`。
5. 进入“风险驾驶舱”，查看评分、中心排名、雷达图。
6. 进入“CAPA审核”，查看CAPA质量评分和补充建议。
7. 进入“核查问答”，生成访谈问答。
8. 进入“任务清单”，生成核查前任务。
9. 进入“报告导出”，生成Word综合报告。

## 当前限制

- V2仍为本地SQLite版本，不适合作为正式多客户生产数据库。
- 默认账号仅用于演示，正式版应接入真实用户管理和密码重置机制。
- PDF仅支持文本型PDF，扫描件需要OCR。
- AI接口为预留配置，尚未真实调用模型。
- 文件存储为本地 uploads 目录，正式版应接入对象存储或私有文件服务器。
